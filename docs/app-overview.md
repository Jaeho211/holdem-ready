# 앱 개요

## 목적

`Holdem Ready`는 라이브 홀덤 입문자가 짧은 문제 풀이와 체크리스트를 통해 실전 감각을 익히도록 만든 모바일 우선 학습 앱입니다.

핵심은 두 가지입니다.

- 의사결정 문제를 자주 반복해서 풀게 한다.
- 틀린 문제와 약한 태그를 바로 다시 연습하게 한다.

## 제품 성격

- 클라이언트 전용 앱
- 로그인 없음
- 서버 API 없음
- 로컬 저장 기반 MVP
- 모바일 화면 폭에 맞춘 단일 페이지 앱

앱 엔트리는 [`app/page.tsx`](../app/page.tsx)이고, 실제 상태 로직은 [`app/_components/holdem-ready-app.tsx`](../app/_components/holdem-ready-app.tsx), 화면 구성은 [`app/_components/screens.tsx`](../app/_components/screens.tsx), 공용 UI는 [`app/_components/ui.tsx`](../app/_components/ui.tsx)에 나뉘어 있습니다.

## 주요 뷰

앱 내부 뷰는 아래 6개입니다.

- `home`
- `training`
- `wrongs`
- `records`
- `liveTips`
- `quiz`

하단 탭은 `home`, `training`, `wrongs`, `records` 네 개만 노출되고, `liveTips`와 `quiz`는 내부 흐름으로 진입합니다.

## 화면별 역할

### 1. 홈

홈은 대시보드 역할을 합니다.

- 오늘 푼 문제 수와 목표 진행률 표시
- 오늘의 10문제 세션 시작 또는 이어풀기
- 현재 약점 태그 노출
- 약점 드릴 5문제 진입
- 라이브 팁 체크 진행도 확인
- 카테고리별 빠른 시작 버튼

### 2. 훈련

훈련 화면에서는 카테고리별 세션을 시작합니다.

- 프리플랍
- 포스트플랍
- 확률
- 라이브 카지노 팁

각 훈련 카드에서 이어풀기와 새로 시작을 선택할 수 있습니다.

### 3. 오답노트

오답 화면은 최근에 틀린 문제를 다시 보는 용도입니다.

- 전체 / 프리플랍 / 포스트플랍 / 확률 필터
- 오답 10문제 재도전
- 문제별 내 선택과 정답 비교
- 태그를 눌러 약점 드릴 시작

라이브 팁은 문제형이 아니라 체크리스트형이라 오답노트에는 쌓이지 않습니다.

### 4. 기록

기록 화면은 학습 통계를 보여줍니다.

- 전체 정답률
- 연속 학습 일수
- 누적 풀이 수
- 카테고리별 정확도
- 최근 7일 학습 빈도
- 약점 3개 추천

### 5. 라이브 팁

라이브 팁은 실전 현장 적응을 위한 체크리스트입니다.

- 핸드 진행 순서
- 핵심 용어
- 빠른 오즈 계산
- 블라인드와 베팅 구조
- 초보 실수

각 항목은 체크 여부만 저장합니다.

### 6. 퀴즈

퀴즈 화면은 세션 진행 화면입니다.

- 문제 본문과 상황 정보 표시
- 보드 5칸과 히어로 핸드를 포커 테이블 형태로 표시
- 확률 카테고리 중 히어로 카드가 없는 문제는 카드 뒷면으로 숨김 표시
- 선택지 버튼 표시
- 정답 여부 피드백
- 해설과 초보자 실수 포인트 제공
- 다음 문제 또는 세션 결과 보기

카드 UI는 작은 모바일 화면에서도 읽기 쉽게 앞면 인덱스와 중앙 수트 심볼을 직접 렌더링합니다.

## 데이터 구성

[`lib/training-data.ts`](../lib/training-data.ts)는 공개 export 진입점이고, 실제 데이터 본문은 `lib/training-data/` 하위 모듈에 나뉘어 있습니다.

- 프리플랍 31문제
- 포스트플랍 29문제
- 확률 28문제
- 라이브 팁 5개 섹션 / 16개 항목

카테고리 메타와 라이브 팁도 같은 디렉터리 안에서 분리 관리합니다.

## 세션 규칙

세션은 모두 랜덤 셔플 기반으로 생성됩니다.

- 오늘 세션: 전체 문제 중 10문제
- 프리플랍 세션: 8문제
- 포스트플랍 세션: 6문제
- 확률 세션: 5문제
- 약점 드릴: 해당 태그 중심 5문제
- 오답 재도전: 최근 오답 중심 최대 10문제

세션 진행 중 상태는 저장되며, 이어풀기가 가능합니다.

## 저장되는 상태

앱은 `localStorage`의 `holdem-ready:v1` 키에 아래 상태를 저장합니다.

- `settings`
- `responses`
- `tipChecks`
- `sessions`

### settings

- 언어
- 진동 여부
- 사운드 여부
- 하루 목표 문제 수

### responses

문제 풀이 이력입니다.

- 문제 ID
- 카테고리
- 내가 고른 선택
- 정답 여부
- 정답 선택지
- 응답 시각
- 태그

### tipChecks

라이브 팁 체크리스트 항목의 완료 여부입니다.

### sessions

진행 중인 세션의 문제 목록, 현재 인덱스, 결과 기록을 보관합니다.

## 통계 계산 방식

기록 화면과 홈에서 쓰는 값은 대부분 클라이언트에서 즉시 계산합니다.

- 전체 정답률
- 카테고리별 정답률
- 최근 40개 오답 기준 약점 태그
- 최근 7일 학습량 추이
- 연속 학습 일수

즉, 별도 집계 서버 없이 브라우저 내 기록만으로 통계를 계산합니다.

## PWA / 오프라인

앱은 기본적인 PWA 구성을 갖고 있습니다.

- [`app/manifest.ts`](../app/manifest.ts)
- [`public/sw.js`](../public/sw.js)

서비스 워커는 앱 셸에 가까운 리소스를 캐시하고, 네트워크 실패 시 캐시를 우선 사용합니다.

## 스타일과 UI 방향

- 모바일 우선
- 최대 폭 `520px`
- 포커 테이블 느낌의 녹색 / 금색 톤
- 카드형 레이아웃과 타원형 테이블 씬
- 앞면 카드는 컴포넌트에서 직접 그린 인덱스/수트 기반 디자인
- 카드 뒷면은 정적 SVG 에셋 사용
- 정답/오답에 따른 진동과 사운드 피드백 옵션

## 현재 구조상 특징

- 구현 속도를 우선한 단일 페이지 구조입니다.
- App Router 엔트리는 얇고, 클라이언트 상태 관리와 화면 UI는 `app/_components/` 아래로 분리돼 있습니다.
- 카드 렌더링은 [`app/_components/ui.tsx`](../app/_components/ui.tsx)와 [`lib/holdem/cards.ts`](../lib/holdem/cards.ts)를 중심으로 관리합니다.
- 서버 연동 없이도 바로 실행 가능한 데모 성격이 강합니다.
